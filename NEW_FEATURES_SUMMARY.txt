   • Highlights obfuscation and homoglyph attacks
   • Analyzes attack method and protocol

   • Auto-detect ISP from IP geolocation data
   • Generate formatted incident report emails
   • Support for 7+ major ISPs (Comcast, Verizon, AT&T, etc.)
   • Backend API integration (configurable endpoint)
   • Failed report retry logic (3x hourly)
   • Correlation IDs for tracking
   • Email template system for manual sending

6. THREAT HISTORY VIEWER
   • Professional web UI for threat analysis
   • Dashboard with live statistics
   • Advanced filtering system (type, severity, status, IP)
   • Full-text search across 500+ threat logs

   • CSV export for compliance and reporting
   • Dark mode support and mobile responsive design

USER WORKFLOWS
================================================================================

WORKFLOW 1: Block & Continue Safely
  1. User visits malicious URL
  2. Extension blocks access, shows block page
  3. Block page displays threat severity and source
  4. User clicks "⚠️ Continue Anyway (Not Recommended)"
  5. Strong warning dialog displayed with risks listed
  6. User confirms they want to proceed
  7. Threat marked as "resumed" in database
  8. Original URL opened for user
  9. User action logged for future analysis

WORKFLOW 2: Analyze Threat History
  1. Click "📊 History" button in extension popup
  2. Open threat history viewer in new tab
  3. View dashboard with threat statistics
  4. Apply filters (attack type, severity, status, search)
  5. Click threat to view full details
  6. See attack patterns, obfuscation methods, IP info
  7. Check if threat already reported to ISP
  8. Export threat list as CSV if needed

WORKFLOW 3: Report Threat to ISP
  1. Identify suspicious threat in history viewer
  2. Click threat to open detail view
  3. Click "📧 Report to ISP" button
  4. System generates incident report
  5. Detects ISP from IP geolocation
  6. Sends report to configured backend or generates email
  7. User can copy email template and send manually
  8. Report status tracked in database (pending/sent/failed)
  9. Failed reports automatically retry hourly

WORKFLOW 4: Export & Share Data
  1. Open threat history viewer
  2. Apply desired filters to show relevant threats
  3. Click "📥 Export" button
  4. CSV file downloaded with complete threat data
  5. Can be imported into Excel, security tools, reports
  6. Includes timestamp, URL, IP, location, severity, status

================================================================================
DATABASE ENHANCEMENTS
================================================================================

New Table: threatLogs
  Stores complete information about each detected threat:
  - id (PRIMARY KEY)
  - timestamp (INDEXED - for sorting)
  - ipAddress (INDEXED - for IP queries)
  - attackType (INDEXED - for type filtering)
  - blocked, resumed, reportedToISP (INDEXED - for status)
  - url, ipLocation, attackStructure, indicators, metadata
  - userAgent, referrer, tabId

New Table: ispReports
  Tracks incident reports sent to ISPs:
  - id (PRIMARY KEY)
  - timestamp (INDEXED - for sorting)
  - threatLogId (INDEXED - for linking)
  - status: 'pending'|'sent'|'failed' (INDEXED)
  - ispEmail, reportData, response

New Database Methods:
  - addThreatLog(log) - Create threat log
  - getThreatLogs(limit) - List all logs
  - getThreatLogById(id) - Get single log
  - updateThreatLog(id, updates) - Mark as resumed/reported
  - getThreatLogsByIP(ipAddress) - Query by IP
  - getThreatLogsByAttackType(type) - Query by type
  - clearOldThreatLogs(days) - Auto cleanup
  - addISPReport(report) - Create report
  - updateISPReport(id, updates) - Update status
  - getPendingISPReports() - Retry logic
  - getISPReports(limit) - List reports

API INTEGRATIONS
================================================================================

1. IP Geolocation API (ipapi.co)
   - Endpoint: https://ipapi.co/{ip}/json/
   - Rate Limit: 1000 requests/day (free)
   - Cache Duration: 24 hours
   - Response: city, country, ISP, ASN, timezone, coordinates

2. ISP Report Backend API (configurable)
   - Endpoint: POST {endpoint}/api/isp-reports
   - Headers: Content-Type: application/json
   - Body: threat data, ISP email, evidence
   - Response: success status, report ID
   - Retry Logic: 3 attempts over 1 hour

SECURITY FEATURES
================================================================================

✅ Data Privacy
   - All URL analysis done locally (no external scanning)
   - IP geolocation cached locally (minimal API calls)
   - No tracking or analytics
   - User data never leaves device except for ISP reports (optional)

✅ Input Sanitization
   - All URLs escaped in history viewer (XSS prevention)
   - HTML entity encoding for threat data
   - No eval() or unsafe DOM manipulation
   - Content Security Policy headers

✅ Report Security
   - ISP reports only sent with explicit user permission
   - Failed reports stored locally (not auto-sent)
   - Correlation IDs prevent duplicate sends
   - User can review email templates before sending

✅ Data Retention
   - Configurable log retention (default 30 days)
   - Automatic cleanup of old logs
   - Manual export before deletion
   - No cloud storage or remote backups
